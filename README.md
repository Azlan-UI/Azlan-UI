## 🧭 About Me

I'm a Computer Science student focused on building things that actually work in production — not just demos. My core interest is in **AI Engineering**: designing systems around large language models that are reliable, retrieval-aware, and grounded in real data rather than guesswork.

Alongside that, I care deeply about **backend engineering and software architecture** — because an LLM feature is only as good as the API, database, and system design supporting it. I'd rather spend an extra hour designing a clean schema or a sane API contract than patch a fragile one later.

I'm a continuous learner by necessity, not habit — this field moves fast, and I treat staying current with RAG techniques, agentic patterns, and system design as part of the job, not extra credit. What drives me is solving real problems: the kind where the constraints are messy and the "correct" answer isn't in a textbook.
